Primary AWS Services

This post covers key AWS services commonly used to build scalable, reliable cloud infrastructure. Below you'll find an overview of each service with its primary purpose and benefits.

Core Services

AWS Identity and Access Management (IAM)

IAM is a web service that helps you securely control access to AWS resources. You use IAM to control who is authenticated (signed in) and authorized (has permissions) to use resources.

When you create an AWS account, you begin with one sign-in identity that has complete access to all AWS services and resources in the account (the root user). AWS strongly recommends not using the root user for everyday tasks, instead safeguarding credentials and delegating with IAM roles and policies.

Amazon EC2 (Elastic Compute Cloud)

Provides scalable computing capacity in the AWS Cloud. You can launch virtual servers as needed, configure security and networking, and manage storage. EC2 enables you to scale up or down to handle changes in requirements, reducing the need to forecast traffic.

Amazon ECS (Elastic Container Service)

A highly scalable, fast container management service. Run, stop, and manage Docker containers on a cluster. ECS supports AWS Fargate (serverless infrastructure) and EC2-backed clusters for greater control.

Amazon EKS (Elastic Kubernetes Service)

A managed service to run Kubernetes without needing to install, operate, and maintain your own control plane or nodes. EKS automates Kubernetes deployment, scaling, and management for containerized applications.

Amazon ECR (Elastic Container Registry)

Secure, scalable, reliable registry for container images. Supports private repositories with IAM-based permissions. Push, pull, and manage Docker and OCI-compatible images with your preferred CLI.

Amazon S3 (Simple Storage Service)

Object storage with industry-leading scalability, data availability, security, and performance. Store and protect any amount of data for data lakes, websites, mobile apps, backup, archive, and analytics.

Amazon EBS (Elastic Block Store)

Provides block-level storage volumes for use with EC2 instances. EBS volumes act like raw, unformatted block devices. They persist independently from the life of an instance and can be resized and reconfigured.

Amazon EFS (Elastic File System)

Simple, serverless, elastic file storage that scales automatically as you add or remove files. Built to support AWS services and on-premises resources with a straightforward web interface for management.

Amazon RDS (Relational Database Service)

Makes it easier to set up, operate, and scale relational databases in AWS. Offers cost-efficient, resizable capacity while managing database administration tasks such as backups, patching, and monitoring.

Amazon VPC (Virtual Private Cloud)

Enables you to provision a logically isolated section of the AWS Cloud where you can launch AWS resources in a defined virtual network. Offers full control over IP addressing, subnets, route tables, and network gateways.

Elastic Load Balancing (ELB)

Automatically distributes incoming traffic across multiple targets like EC2 instances, containers, and IP addresses in one or more Availability Zones. Monitors health and routes traffic only to healthy targets.

Amazon Route 53

A highly available and scalable DNS web service. Supports domain registration, DNS routing, and health checking to ensure availability and reliability for your applications.

Amazon CloudWatch

Monitors AWS resources and applications in real time. Collects and tracks metrics, logs, and events. Enables you to create alarms, visualize data with dashboards, and respond automatically to changes in resource usage.

CloudWatch provides system-wide visibility into resource utilization, application performance, and operational health to maintain high availability and reliability.
