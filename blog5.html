Primary AWS Services

By Dee Sanas

This post covers primary AWS services (EC2, S3, EBS, EFS, RDS, VPC, ELB, Route53, IAM, and CloudWatch) along with other key services you should know about.

  • Amazon Elastic Compute Cloud (Amazon EC2)
  • Amazon Elastic Container Service (Amazon ECS)
  • Amazon Elastic Kubernetes Service (Amazon EKS)
  • Amazon Elastic Container Registry (Amazon ECR)
  • Amazon Simple Storage Service (Amazon S3)
  • Amazon Elastic Block Store (Amazon EBS)
  • Amazon Elastic File System (Amazon EFS)
  • Amazon Relational Database Service (Amazon RDS)
  • Amazon Virtual Private Cloud (Amazon VPC)
  • Amazon Elastic Load Balancing (ELB)
  • Amazon Route 53
  • Amazon CloudWatch
  • AWS Identity and Access Management (IAM)

IAM

AWS Identity and Access Management (IAM) is a web service that helps you securely control access to AWS resources. You use IAM to control who is authenticated (signed in) and authorized (has permissions) to use resources. You begin with one sign-in identity (the root user) with full access to all resources, but you should avoid using this for daily work.

Amazon EC2

Provides scalable computing capacity in the AWS Cloud. Eliminates the need to invest in hardware up front, so you can develop and deploy applications faster. Lets you launch, configure, and manage virtual servers as needed.

Amazon ECS

Highly scalable, fast container management service. Lets you run, stop, and manage containers on a cluster, with integration for AWS Fargate for serverless infrastructure or EC2 for self-managed clusters.

Amazon EKS

Managed service for running Kubernetes on AWS without the need to maintain your own control plane or nodes. Supports automated scaling and management of containerized applications.

Amazon ECR

Fully managed container image registry supporting Docker and OCI images. Integrated with IAM for secure, scalable access. Supports CLI for pushing, pulling, and managing images.

Amazon S3

Object storage service offering industry-leading scalability, availability, security, and performance. Ideal for storing any amount of data for a variety of use cases including data lakes, backup and restore, and big data analytics.

Amazon EBS

Provides block-level storage volumes for EC2 instances. Behaves like raw, unformatted block devices. You can attach, format, and mount them as you would a traditional disk, with persistence beyond the life of an EC2 instance.

Amazon EFS

Serverless, elastic file storage for AWS Cloud services and on-premises resources. Scales automatically as you add and remove files, eliminating the need to provision or manage capacity.

Amazon RDS

Managed relational database service that makes it easier to set up, operate, and scale databases in the AWS Cloud. Supports multiple engines and automates backups, patching, and scaling.

Amazon VPC

Lets you launch AWS resources into a virtual network you've defined. Offers full control over networking configuration, including subnets, routing tables, and security settings, resembling a traditional data center network.

Elastic Load Balancing (ELB)

Automatically distributes incoming traffic across multiple targets (EC2 instances, containers, IP addresses) in one or more Availability Zones. Monitors target health and routes traffic to healthy instances only.

Amazon Route 53

Highly available, scalable DNS web service that offers domain registration, DNS routing, and health checking. Helps route users to endpoints reliably and cost-effectively.

Amazon CloudWatch

Monitors AWS resources and applications in real time. Collects and tracks metrics, logs, and events. Lets you set alarms, visualize data with dashboards, and automate responses to changes in resource utilization or performance.
